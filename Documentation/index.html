Wacky Tetris

This is a version of Tetris that runs on macOS and iOS using the same code base for everything except the UI and parts of the game views. The macOS version is more capable (which is to be expected, given the host OS and hardware) but the iOS version allows for more interactivity by the user.

Wacky Tetris is an experiment and continuation of a test program I wrote for the Nintendo 64 (as a debug target for debugging hardware) in the late 1990s.

Current Feature Set

Wacky Tetris has the current features:

  • Standard Tetris playing pieces.
  • Board of 10x20 in size.
  • Single player mode with the following motions:
    • Left (the piece is moved one column to the left)
    • Rotate left (the piece is rotated leftwards, eg, counter-clockwise)
    • Right (the piece is moved one column to the right)
    • Rotate right (the piece is rotated rightwards, eg, clockwise)
    • Down (the piece is moved down)
    • Drop down (the piece is dropped down as far as it can go then frozen into place)
    • Up (the piece is moved up)
    • Up and Away (the piece is discarded, visually, the piece zooms away to the top)
  • AI/Attract mode.
  • Multiple piece sets (but only the standard set is active right now).

Planned Feature Set

The following features are planned for Wacky Tetris:

  • Finalize AI such that if works better (as defined by very few gaps when it plays). Implement sneak peak feature of AI (to let the AI look at more than one piece to find the best score).
  • Finalize level implementation to allow for levels (eg, easy, hard, etc) as well as user-defined levels.
  • Change board sizes as per the user.
  • Finalize scoring methodology.
  • 3D version of Tetris. The board will remain 2D but be in a 3D world.
  • Multiplayer - two (or more?) players use local network to play against each other.
  • Add iOS-specific features, such as turning the device upside down to stop a game (and the blocks fall out) and the like.
  • Add sound effects and music (need to find public domain sound and music…).
  • Add the wacky to Wacky Tetris:
    • Add action buttons. There are two general types of action buttons:
    • Good action buttons that help the user (remove rows, add to score, etc).
    • Dangerous action buttons that make life difficult for the user.
    • Board rotations along multiple axes to confuse the user.
    • Board resizing on the fly.
    • Picture mode where retired pieces for a picture (ala masking the foreground to see the picture behind the bucket).

Current Status

Wacky Tetris isn’t very wacky at the moment… Only 2D is implemented.

Plans
